Introduction

  • Ruby is a pure object-oriented programming language. It was created in 1993 by Yukihiro Matsumoto of Japan.
Features of Ruby
  • Ruby is a general-purpose, interpreted programming language.
  • Ruby is a server-side scripting language similar to Python and PERL.
  • Ruby can be embedded into Hypertext Markup Language (HTML).
  • Ruby has a rich set of built-in functions, which can be used directly into Ruby scripts.

Ruby Operators

  • Ruby Arithmetic Operators +,-,/,%,*
  • Exponent is Represented as **     e.g   2**3 = 8
  • a<=>b

    Combined comparison operator Returns 0 if a equals b, 1 if a is greater than b and -1 if a is less than b.

  • === :-

    Used to test equality within a when clause of a case statement.  e.g   { (1...100) === 51 } returns true

  • .eql? :-

    True if the receiver and argument have both the same Data type and equal values.   e.g   { 1.eql?(1.0) }

  • Ruby parallel Assignment   e.g   { a, b, c = 10, 20, 30 }
  • Ruby Logical Operators   and,or,not,&&,||,!
  • Ruby Range operators (1..10) Creates a range from 1 to 10 inclusive, whereas (1...10) 1...10 Creates a range from 1 to 9

Variables in a Ruby Class

Ruby provides four types of variables:


  • Local Variables: Local variables are the variables that are defined in a method.Local variables begin with a lowercase letter or _.

  • Instance Variables: Instance variables are available across methods for any particular instance or object.Instance variables are preceded by the at sign (@) followed by the variable name.

  • Class Variables: Class variables are available across different objects. They are preceded by the sign @@ and are followed by the variable name.

  • Global Variables: Class variables are not available across classes.The global variables are always preceded by the dollar sign ($)

Access Specifiers

Public:methods can be called by everyone - no access control is enforced

Protected:methods can be invoked only by objects of the defining class and its subclasses.

Private:methods can be called only in the context of the current object; you cannot invoke another object's private methods.

Lambda

Lambda is like a function but without a name , in ruby lamdas are also objects.

l = lambda do |string|
  if string == "try"
    return "There's no such thing"
  else
    return "Do or do not."
  end
end
puts l.call("try")

difference between block,proc and lambda

  • proc and lambda are objects whereas block is not an object.
  • Only one block can be passed as arguement , in contrast we can pass multiple procs as arguments.
  • Lambdas check the number of arguments, while procs do not.
  • Lambdas and procs treat the ‘return’ keyword differently.

    Exception

    Ruby provide a nice mechanism to handle exceptions. We enclose the code that could raise an exception in a begin/end block and use rescue clauses to tell Ruby the types of exceptions we want to handle. 
    begin
   file = open("/unexistant_file")
   if file
      puts "File opened successfully"
   end
rescue
      puts "File opened successfully"
end

    Retry keyword

    You can capture an exception using rescue block and then use retry statement to execute begin block from the beginning. 
        begin
   file = open("/unexistant_file")
   if file
      puts "File opened successfully"
   end
rescue
   fname = "existant_file"
   retry
end
